Project rules:

- always use TypeScript strict mode
- use Prisma for database
- follow NestJS module structure
- avoid business logic in controllers

### Database naming convention
- Use snake_case and lowercase for all database identifiers.
- Table names must be plural.
- Column names must also use snake_case.
- Foreign keys should follow the pattern <referenced_table>_id.
Examples:
- Tables: `users`, `contacts`, `deals`
- Columns: `created_at`, `organization_id`
- Use `BIGINT GENERATED BY DEFAULT AS IDENTITY` for primary keys.
- The primary key column must be named `id`.
- IDs are incremental and start at 1.
